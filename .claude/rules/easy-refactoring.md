# Easy Refactoring Guide (SHOULD)

리팩토링 작업 시 준수해야 할 가이드

## 핵심 원칙

### 1. Declarative 관점 유지

디자이너 입장에서 각 컴포넌트가 어떻게 작동되는지 철저히 declarative한 입장에서 관련 함수 리팩토링 및 주석을 수정한다.

### 2. 사용자 관점 주석

주석에는 동작 방식과 단계가 **사용자의 인풋**, **시각적 변화** 관점에서 서술한다.

```jsx
/**
 * 슬라이더 컴포넌트
 *
 * 동작 흐름:
 * 1. 사용자가 슬라이더 핸들을 드래그한다
 * 2. 핸들 위치에 따라 값이 0-100 사이로 변경된다
 * 3. 값 변경 시 라벨에 현재 값이 실시간으로 표시된다
 * 4. 드래그 종료 시 onChange 콜백이 호출된다
 */
```

### 3. 기능/로직/형태 보존

**절대로** 기능과 기본 로직, 형태가 바뀌지 않도록 조심한다.

## 체크리스트

리팩토링 전:
- [ ] 외부 동작이 변경되지 않음을 확인
- [ ] 기존 스토리가 모두 통과하는지 확인

리팩토링 후:
- [ ] 주석이 사용자 관점으로 작성되었는지 확인
- [ ] 기존 스토리가 여전히 통과하는지 확인
- [ ] 시각적 변화가 없는지 확인
