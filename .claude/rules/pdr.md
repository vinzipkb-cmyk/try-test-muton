# Product Design Requirements (PDR)

## 1. 제품 정의

### 1.1 제품명
MUTON Brand Website

### 1.2 한 줄 정의
하이엔드 주방가구 브랜드의 건축적 정체성을 전달하는 아카이브 중심 브랜드 사이트

### 1.3 핵심 가치 제안
- "가구 쇼핑몰"이 아닌 "건축적 작품 아카이브"
- 증거 기반 신뢰 구축 (디테일, 소재, 공정)
- 서사적 확신을 통한 문의 전환

---

## 2. 타깃 사용자 & 시나리오

### 2.1 사용자 세그먼트

| ID | 세그먼트 | 특징 | 핵심 니즈 |
|----|----------|------|----------|
| U1 | 오너 (감각형) | 초고자산가, 주거/리조트 오너 | "이 브랜드가 뭐가 다른가?" |
| U2 | 디벨로퍼 (사업형) | 호텔/리조트 운영사, 건축사무소 | "품질, 납기, 리스크는?" |
| U3 | 실무자 (스펙형) | 인테리어 디렉터, PM, 자재 담당 | "스펙 확인, 샘플 요청" |

### 2.2 사용자 시나리오

#### S1. 오너의 브랜드 탐색
```
진입 → 브랜드 인상 형성 → 작품 탐색 → 디테일 확인 → 확신 → 문의
```

| 단계 | 사용자 행동 | 기대 경험 | 필요 콘텐츠 |
|------|------------|----------|------------|
| 1. 진입 | 랜딩 페이지 도착 | 강렬한 첫인상 | Hero 영상/이미지 |
| 2. 인상 | 스크롤하며 훑어봄 | "다르다" 느낌 | 대표 프로젝트 미리보기 |
| 3. 탐색 | Archive 진입 | 작품처럼 감상 | 프로젝트 그리드 |
| 4. 확인 | 프로젝트 상세 진입 | 디테일에 감탄 | 고해상도 이미지, 소재 정보 |
| 5. 확신 | 여러 프로젝트 비교 | 일관된 퀄리티 확인 | 관련 프로젝트 추천 |
| 6. 문의 | Contact 이동 | 쉬운 연락 | 간결한 문의 폼 |

#### S2. 디벨로퍼의 검증 여정
```
진입 → 레퍼런스 확인 → 역량 검증 → 프로세스 이해 → 미팅 요청
```

| 단계 | 사용자 행동 | 기대 경험 | 필요 콘텐츠 |
|------|------------|----------|------------|
| 1. 진입 | 직접 URL 또는 검색 | 전문적 인상 | 깔끔한 네비게이션 |
| 2. 레퍼런스 | Archive에서 유사 프로젝트 검색 | 관련 사례 빠른 발견 | 필터 (타입, 규모) |
| 3. 역량 | About/Lab 확인 | 제작 역량 파악 | 프로세스, 소재 연구 |
| 4. 프로세스 | About 상세 읽기 | 협업 방식 이해 | 단계별 프로세스 |
| 5. 미팅 | Contact 이동 | 비즈니스 문의 | 문의 유형 선택 |

#### S3. 실무자의 스펙 확인
```
진입 → 프로젝트/소재 검색 → 상세 스펙 확인 → 자료 요청
```

| 단계 | 사용자 행동 | 기대 경험 | 필요 콘텐츠 |
|------|------------|----------|------------|
| 1. 진입 | 특정 프로젝트/소재 검색 | 빠른 접근 | 검색, 필터 |
| 2. 검색 | Lab에서 소재 탐색 | 체계적 분류 | 카테고리 필터 |
| 3. 스펙 | 상세 정보 확인 | 기술 사양 확인 | 스펙 테이블, 이미지 |
| 4. 요청 | 자료/샘플 요청 | 간편한 요청 | 문의 폼 (자료 요청 옵션) |

## 4. 페이지별 요구사항

### 4.1 Landing Page

#### 목적
첫인상 형성, 브랜드 정체성 전달

#### 필수 요소
| 요소 | 설명 | 우선순위 |
|------|------|----------|
| Hero | 전체화면 영상/이미지 + 한 문장 | P0 |
| Project Preview | 대표 프로젝트 3-6개 그리드 | P0 |
| Lab Preview | 소재/연구 미리보기 | P1 |
| CTA | Archive, Contact 유도 (은은하게) | P1 |

#### 콘텐츠 요구사항
- Hero 영상: 무음, 자동재생, 루프
- Hero 카피: 1문장 (10단어 이내)
- 프로젝트 카드: 이미지 + 호버 시 제목/연도

### 4.2 About Page

#### 목적
브랜드 신뢰 구축, 역량 증명

#### 필수 요소
| 요소 | 설명 | 우선순위 |
|------|------|----------|
| Philosophy | 브랜드 철학 (짧게) | P0 |
| Process | 설계-제작-설치 프로세스 | P0 |
| Partners | 협력사/클라이언트 로고 | P1 |
| Team | 팀 소개 (선택) | P2 |

#### 콘텐츠 요구사항
- 철학: 3문장 이내
- 프로세스: 3-5단계, 각 단계 1문장 + 이미지
- 파트너: 로고만 (텍스트 최소)

### 4.3 Archive Page

#### 목적
프로젝트 아카이브 탐색, 작품으로서의 감상

#### 필수 요소 - 리스트
| 요소 | 설명 | 우선순위 |
|------|------|----------|
| Grid | 1px 간격 프로젝트 그리드 | P0 |
| Filter | 타입, 연도, 규모 필터 | P1 |
| Hover Info | 프로젝트명, 연도, 키워드 | P0 |

#### 필수 요소 - 상세
| 요소 | 설명 | 우선순위 |
|------|------|----------|
| Hero Image | 대표 이미지 (전체폭) | P0 |
| Project Info | 제목, 위치, 연도, 규모, 타입 | P0 |
| Description | 프로젝트 설명 (3-5문장) | P0 |
| Gallery | 이미지 갤러리 (순서대로) | P0 |
| Materials | 사용 소재 목록 | P1 |
| Related | 관련 프로젝트 추천 | P1 |

### 4.4 Lab Page

#### 목적
소재/빛/디테일 연구 아카이브, 전문성 증명

#### 필수 요소 - 리스트
| 요소 | 설명 | 우선순위 |
|------|------|----------|
| Grid | 연구 주제 그리드 | P0 |
| Category Filter | 소재, 빛, 하드웨어 등 | P0 |
| Hover Info | 주제명, 카테고리 | P0 |

#### 필수 요소 - 상세
| 요소 | 설명 | 우선순위 |
|------|------|----------|
| Hero Image | 대표 이미지 | P0 |
| Topic Info | 제목, 카테고리 | P0 |
| Content | 연구 내용 (이미지 + 텍스트) | P0 |
| Related Projects | 이 소재/기술 적용 프로젝트 | P1 |

### 4.5 Contact Page

#### 목적
문의 전환, 미팅 예약

#### 필수 요소
| 요소 | 설명 | 우선순위 |
|------|------|----------|
| Inquiry Form | 문의 폼 | P0 |
| Contact Info | 이메일, 전화, 주소 | P0 |
| Inquiry Type | 문의 유형 선택 | P1 |

#### 폼 필드
| 필드 | 타입 | 필수 |
|------|------|------|
| 이름 | text | O |
| 이메일 | email | O |
| 연락처 | tel | X |
| 문의 유형 | select | O |
| 메시지 | textarea | O |

#### 문의 유형 옵션
- 프로젝트 상담
- 자료 요청
- 파트너십 문의
- 기타

## 6. 비기능 요구사항

### 6.1 성능
- First Contentful Paint: < 1.5s
- 이미지 최적화: WebP, lazy loading
- 비디오: 압축, 스트리밍

### 6.2 접근성
- 키보드 네비게이션
- 이미지 alt 텍스트
- 충분한 색상 대비

### 6.3 SEO
- 시맨틱 HTML
- 메타 태그 (OG, Twitter)
- 구조화된 데이터

---

## 7. 제약 조건

### 7.1 브랜드 제약
- 자기선언형 수식어 금지 (최고급, 프리미엄)
- 과도한 애니메이션 금지
- border-radius: 0 (각진 형태)
- 모노톤 기반 컬러

### 7.2 기술 제약
- React + MUI 기반
- Storybook 문서화 필수
- 정적 사이트 (CMS 없음, 데이터는 정적)
